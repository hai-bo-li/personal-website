# 个人网站部署指南

最简单的方案：**GitHub + Vercel**，免费、自动 HTTPS、国内访问速度还可以。

---

## 方案一：Vercel 部署（推荐 ⭐⭐⭐⭐⭐）

### 第一步：把网站文件传到 GitHub

1. 去 [github.com](https://github.com) 注册/登录账号
2. 新建一个仓库（New Repository），名字比如叫 `personal-website`，选 Public
3. 把你本地的 `个人网站` 文件夹里的所有文件上传到这个仓库

   仓库结构应该是这样的：
   ```
   personal-website/
   ├── index.html
   └── articles/
       └── ai-platforms-comparison.html
   ```

   > 注意：`index.html` 要在仓库根目录，不能套在子文件夹里

### 第二步：Vercel 一键部署

1. 去 [vercel.com](https://vercel.com)，用 GitHub 账号登录
2. 点击 **Add New → Project**
3. 选择你刚创建的 `personal-website` 仓库，点 **Import**
4. 配置页面什么都不用改，直接点 **Deploy**
5. 等 1-2 分钟，部署完成！Vercel 会给你一个免费域名，类似 `your-project.vercel.app`

### 第三步：后续更新

以后修改了网站内容，只要 push 到 GitHub，Vercel 会**自动重新部署**，不用手动操作。

---

## 方案二：GitHub Pages（纯 GitHub）

如果你不想用 Vercel，GitHub 自带的 Pages 也能用：

1. 仓库 → Settings → Pages
2. Source 选 `Deploy from a branch`，Branch 选 `main` / `(root)`
3. 等几分钟，访问 `https://你的用户名.github.io/personal-website/`

> ⚠️ 注意：GitHub Pages 国内访问速度比 Vercel 慢一些，而且默认是子路径（/personal-website/），需要调整一下 html 里的链接路径（把 `../index.html` 改成绝对路径）。

---

## 方案三：国内更快的选择

如果后面觉得 Vercel 国内访问不够快，可以考虑：

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Vercel** | 部署最简单，免费，海外访问快 | 国内速度中等 |
| **Cloudflare Pages** | 免费，全球 CDN，国内速度比 Vercel 好 | 注册需要手机号 |
| **阿里云 OSS + CDN** | 国内速度最快 | 需要域名备案，有少量费用 |
| **腾讯云 COS + CDN** | 国内速度快 | 需要域名备案，有少量费用 |

个人使用 **Vercel 足够了**，等网站内容多了、有更多访客了再考虑换。

## 绑定自己的域名（可选）

如果有自己的域名（比如 `boshushu.com`），在 Vercel 里设置一下：

1. Vercel 项目 → Settings → Domains
2. 输入你的域名，点 Add
3. 按照提示去域名商那里加一条 CNAME 解析
4. Vercel 会自动配置 HTTPS 证书
